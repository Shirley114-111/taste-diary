# Taste Diary 上线部署说明

目标：让任何人打开一个网址就能用，并且「拍照识别」功能真实可用。

整体结构：
- **后端**（Node 服务，负责识别和报告）→ 部署到 **Render**（免费）
- **前端**（网页界面）→ 部署到 **Vercel**（免费）
- 百度识别密钥 → 只填在 Render 的环境变量里，**绝不放进代码仓库**

全程大约 15–20 分钟。下面每一步都写了点哪里。

---

## ⚠️ 开始前必读：密钥安全

这个项目的 `backend/.env` 文件里有你的百度 API 密钥。

- 打包给你的版本**已经自动排除了 `.env`**，并带了 `.gitignore`，正常操作不会把它传上去。
- 密钥**只填在 Render 后台**（私密），不要写进任何上传到 GitHub 的文件。
- 因为这串密钥已经在压缩包里流转过，建议上线后到百度控制台**重置一次密钥**（生成新的、作废旧的），更安全。重置入口：百度智能云控制台 → 你的应用 → 查看/管理 AppKey。

---

## 第一步：拿到百度识别密钥

如果你已经有了（压缩包里那对），可以跳过，或按上面的建议重置后用新的。

1. 打开 https://cloud.baidu.com/ ，注册并完成实名认证（个人免费）。
2. 控制台 → 人工智能 → 图像识别 → 创建应用，勾选「菜品识别」。
3. 应用创建后，页面上能看到 **API Key** 和 **Secret Key** 两串字符。
4. 把这两串先复制到记事本，第三步要用。免费额度 500 次/天，个人够用。

---

## 第二步：把代码放上 GitHub

> 没有 GitHub 账号就先去 https://github.com 注册一个（免费）。
> 完全不想用 GitHub 也行，Render 和 Vercel 都支持直接拖文件夹上传，见文末「不用 GitHub 的办法」。

1. 在 GitHub 点右上角 **+** → **New repository**，名字随便（比如 `taste-diary`），点 **Create repository**。
2. 把我给你的整个 `taste-diary-deploy` 文件夹里的内容上传上去。最简单的方式：在新仓库页面点 **uploading an existing file**，把文件夹里的东西全拖进去，然后 **Commit changes**。
   - 确认上传的内容里**没有** `backend/.env` 这个文件（应该本来就没有，我已排除）。

---

## 第三步：部署后端到 Render

1. 打开 https://render.com ，用 GitHub 账号登录。
2. 点 **New +** → **Web Service**。
3. 选你刚建的那个仓库，点 **Connect**。
4. 填写设置：
   - **Name**：随便，比如 `taste-diary-backend`
   - **Root Directory**：填 `backend`
   - **Build Command**：`npm install`
   - **Start Command**：`npm start`
   - **Instance Type**：选 **Free**
5. 往下找到 **Environment Variables**，点 **Add Environment Variable**，加这两条（**这里就是填密钥的地方**）：
   - Key 填 `BAIDU_API_KEY`，Value 填你的 API Key
   - Key 填 `BAIDU_SECRET_KEY`，Value 填你的 Secret Key
6. 点 **Create Web Service**，等它部署完（几分钟）。
7. 部署成功后，页面顶部会显示一个网址，形如：
   `https://taste-diary-backend.onrender.com`
   **把这个网址复制下来**，下一步要用。

> 免费版 Render 有个特点：十几分钟没人访问会休眠，下次打开第一下要等几十秒唤醒。个人使用没问题。

---

## 第四步：告诉前端后端在哪

1. 在你电脑上的文件夹里，打开 `frontend/config.js`。
2. 把那一行改成你第三步的后端网址（结尾不要加斜杠）：
   ```js
   window.API_BASE = 'https://taste-diary-backend.onrender.com';
   ```
3. 保存，并把这个改动同步到 GitHub（在 GitHub 上直接编辑这个文件也行：进仓库 → `frontend/config.js` → 点铅笔图标 → 改完 Commit）。

---

## 第五步：部署前端到 Vercel

1. 打开 https://vercel.com ，用 GitHub 账号登录。
2. 点 **Add New** → **Project**，选你的仓库，点 **Import**。
3. 设置里找到 **Root Directory**，点 **Edit**，选 `frontend`。
4. 其它保持默认，点 **Deploy**。
5. 部署完会给你一个网址，形如 `https://taste-diary.vercel.app` —— **这就是你要发给别人的网址。**

---

## 第六步（推荐）：收紧后端，只允许你的前端访问

1. 回到 Render 后端的 **Environment** 设置。
2. 再加一条环境变量：
   - Key 填 `FRONTEND_ORIGIN`，Value 填你第五步的 Vercel 网址（如 `https://taste-diary.vercel.app`）
3. 保存，Render 会自动重新部署。这样只有你的网页能调后端，更安全。

---

## 完成后怎么验证

1. 打开 Vercel 给的网址。
2. 进去拍一张或上传一张食物照片，点识别。
3. 如果能认出食物并生成报告，就成功了。
4. 如果识别一直失败：
   - 先单独打开后端网址加 `/api/status`（如 `https://...onrender.com/api/status`），看 `hasKey` 是不是 `true`。是 `false` 说明 Render 的密钥没填对。
   - 确认 `frontend/config.js` 里的后端网址填对了、没多斜杠。
   - 免费后端可能在休眠，第一次访问多等几十秒再试。

---

## 不用 GitHub 的办法

- **Render**：也能用 CLI 或直接连 GitHub 之外的方式，但最省事还是 GitHub。若坚持不用，可考虑把后端换到支持拖拽的平台，步骤会不同，到时可以再帮你调整。
- **Vercel**：装一个 Vercel CLI 后，在 `frontend` 文件夹里跑 `vercel` 也能上传，不强制 GitHub。
